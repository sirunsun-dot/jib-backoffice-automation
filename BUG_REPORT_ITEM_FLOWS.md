# 🎟 COUPONS — Flow เพิ่มรายการคูปอง (Dialog)

URL: `/store/promotion-manager/coupons/create` → dialog `เพิ่มรายการคูปอง`

## โครงสร้างฟอร์ม
- **ข้อมูลทั่วไป:** ชื่อรายการคูปอง TH/EN + toggle "ใช้เหมือนกัน 2 ภาษา"
- **เงื่อนไขส่วนลด:**
  - Combobox `รูปแบบส่วนลด *`: ส่วนลดทั้งตะกร้า / ส่วนลดค่าสินค้า / ส่วนลดค่าจัดส่ง
  - Radio `ประเภทส่วนลด *`: เปอร์เซ็นต์ (%) / จำนวนเงิน (บาท)
  - `มูลค่าส่วนลด *` (default `0`, placeholder "0 - 100" สำหรับ %)
  - `ส่วนลดสูงสุด` (placeholder ไม่จำกัด)
  - `ยอดซื้อขั้นต่ำ` / `จำนวนชิ้นขั้นต่ำ`
  - Checkbox `ใช้ร่วมกับส่วนลดอื่นได้`
- **รหัสคูปอง:**
  - Radio `รูปแบบรหัส *`: รหัสเดียว (default) / สร้างรหัสอัตโนมัติ
  - ถ้าเลือก "รหัสเดียว" → ช่อง `รหัสคูปอง *` (placeholder "เช่น NEWYEAR2025")
  - ถ้าเลือก "สร้างรหัสอัตโนมัติ" → `คำนำหน้ารหัส *`, `ความยาวส่วนท้าย *`, `ชุดอักขระ *` (ตัวเลข / ตัวอักษร / ทั้งสองอย่าง)
- **การใช้งาน:** `จำนวนการใช้ต่อผู้ใช้ *` (default 1), `จำนวนการใช้ทั้งหมด *` (default 1), `จำนวนการแจกจ่าย` (ไม่จำกัด)
- **สินค้าที่ร่วมรายการคูปอง:** Radio ใช้กับสินค้าทั้งหมด / เลือกเฉพาะสินค้าที่ต้องการ

## 🎯 Top Priority แก้ก่อน

1. **CP-5** ป้องกันรหัสคูปองซ้ำใน scope แคมเปญเดียว — หาก submit ไป backend อาจเกิด race condition / duplicate key error
2. **CP-7** เพิ่ม cross-field validation: `userUsageLimit ≤ usageLimit`
3. **CP-2** ปรับ schema ให้ `discountValue > 0` (validator ปัจจุบันใช้ `>= 0` ผิด)
4. **FG-5** Input "จำนวนขั้นต่ำ (ชิ้น)" ต้องเป็น integer เท่านั้น (`step="1"` + parseInt)
5. **CP-6** รหัสคูปองควรจำกัด regex `^[A-Z0-9_-]+$` หรืออย่างน้อยห้าม whitespace + special punctuation
